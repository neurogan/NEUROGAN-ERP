# R2-03 CAPA / QMS Backbone Design

**Regulation:** 21 CFR Part 111 §111.140  
**Date:** 2026-05-02

---

## Problem

The ERP has module-specific investigation tables (`oosInvestigations`, `complaintInvestigations`, `returnInvestigations`) but no central CAPA register. An FDA inspector asking "show me your open CAPAs" has no single answer. This is the first thing a Part 111 auditor looks for.

---

## Approach

Thin coordinator (Option A): `nonconformances` is a lightweight central register. Existing investigation tables stay as-is. QA manually opens an NC when one is warranted (regulatory judgment, not auto-trigger). CAPAs link to NCs. Start fresh — no retroactive migration of existing records.

---

## Navigation

New **"CAPA"** subtab under the Quality page at `/quality/capa`.

## Business Rules

- NC status flow: OPEN → UNDER_INVESTIGATION → CAPA_OPEN → CLOSED
- CAPA cannot close unless an effectiveness check exists with result ≠ PENDING and `performed_at` ≥ 30 days after last `capa_action.completed_at`
- CAPA open requires Part-11 QA signature (`CAPA_OPEN`)
- CAPA close requires Part-11 QA signature (`CAPA_CLOSE`)
- `nc_number` and `capa_number` are system-assigned sequential per year (cannot be set by user)
- Any authenticated user can open an NC; QA/ADMIN can mutate CAPAs

## API

All routes under `/api/quality/capa`. Auth required on all; mutations require QA or ADMIN except NC creation (any user).

### Nonconformances
- `GET /nonconformances` — list; filterable by `status`, `type`, `severity`
- `POST /nonconformances` — create (any authenticated user)
- `GET /nonconformances/:id` — detail with linked CAPA if exists
- `PATCH /nonconformances/:id/status` — transition status (QA/ADMIN)

### CAPAs
- `POST /capas` — open CAPA against NC; Part-11 signature (`CAPA_OPEN`); transitions NC → CAPA_OPEN
- `GET /capas` — list; filterable by `status`
- `GET /capas/:id` — detail with actions + effectiveness checks
- `POST /capas/:id/close` — Part-11 signature (`CAPA_CLOSE`); validates effectiveness check ≥30 days; transitions NC → CLOSED

### Actions
- `POST /capas/:id/actions` — add action (QA/ADMIN)
- `PATCH /capas/:id/actions/:actionId/complete` — mark completed (QA/ADMIN)

### Effectiveness Checks
- `POST /capas/:id/effectiveness-checks` — schedule or record (QA/ADMIN)
- `PATCH /capas/:id/effectiveness-checks/:checkId` — record result (QA/ADMIN)

### Management Reviews
- `GET /management-reviews` — list
- `POST /management-reviews` — create with capaIds; Part-11 signature (`MANAGEMENT_REVIEW`)
- `GET /management-reviews/:id` — detail

## UI

**Default view — NC List**
- Filter bar: status, type, severity
- Table: NC#, title, type, severity, status badge, created date, linked CAPA# if exists
- "+ Open NC" button (any user) — sheet with title, description, type, severity, optional source link
- Row click → NC detail

**NC Detail**
- Header: NC#, title, type, severity, status badge, description
- Source link (clickable into existing module) if present
- Status transition controls (QA)
- CAPA section: "Open CAPA" (QA, when no CAPA yet) with capaType + rootCause + Part-11 ceremony; or linked CAPA# with navigation

**CAPA Detail**
- Header: CAPA#, type, status, root cause
- Actions table: description, assignee, due, status — "+ Add action", "Mark complete" per row
- Effectiveness check section: schedule (when all actions complete), record result
- "Close CAPA" (QA, enabled when effectiveness check complete and ≥30 days) → Part-11 ceremony

**CAPA List** (toggle from NC list)
- Filter: status
- Table: CAPA#, NC#, type, root cause (truncated), status, next due date

**Management Reviews** (toggle)
- List + "New management review" (QA/ADMIN): period, summary, outcome, select CAPAs to include → Part-11

## Audit Trail

New audit actions: `NC_OPENED`, `NC_STATUS_CHANGED`, `CAPA_OPENED`, `CAPA_ACTION_ADDED`, `CAPA_ACTION_COMPLETED`, `CAPA_EFFECTIVENESS_RECORDED`, `CAPA_CLOSED`, `MANAGEMENT_REVIEW_SIGNED`

New signature meanings: `CAPA_OPEN`, `CAPA_CLOSE`, `MANAGEMENT_REVIEW`
